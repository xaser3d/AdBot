Plugin Details

AdBot

Unreal Engine plugin for Yandex Mobile Ads, AppMetrica analytics, optional AppMetrica Push, and Yandex Mediation on Android.

[ Plugin Banner Image ]

What is AdBot?

AdBot integrates Yandex Mobile Ads SDK 8.0, AppMetrica Analytics 8.2, and optional AppMetrica Push 4.3 (FCM and HMS) into Unreal Engine. Blueprint and C++ cover ads, statistics, and push — no hand-written Java.

Five Yandex Mediation adapters (VK, Mintegral, BIGO, Liftoff, InMobi) are toggled in Project Settings. Drop the plugin into Plugins/AdBot and configure keys and unit IDs under Edit → Project Settings → Plugins → AdBot.

🎯

4 Ad Formats

Banner, Interstitial, Rewarded Video, App Open.

🔵

Blueprint First

Ads use async nodes (Initialize Yandex Ads, Load/Show Banner, …). Privacy, hide banner, and AppMetrica are callable. No C++ required.

🔒

Privacy Controls

User consent, COPPA, location tracking, GDPR-style flags.

📡

5 Mediation Networks

VK, Mintegral, BIGO, Liftoff, InMobi — enable in Project Settings and Yandex cabinet.

📊

AppMetrica 8

Events, profiles, sessions, device ID, crash reporting.

🔔

AppMetrica Push

FCM + HMS, payload delegate, notification permission (Android 13+).

🛠

Debug Panel

Yandex in-app integration checker for development builds.

Rewarded video — use async nodes Load Rewarded Video → Show Rewarded Video. See Rewarded Video and Blueprint example.
Android 13+ notifications — cannot be enabled by default; see POST_NOTIFICATIONS or What changed.

Optional per-request targeting via AdTargeting: age, gender, geo-location, context query, context tags. All configured through Blueprint before loading ads.

AdBot ships AppMetrica Android SDK 8.2.x — this is the mobile analytics product paired with Yandex Ads (not the web Yandex.Metrica counter). Report custom events, user profiles, sessions, and crashes from Blueprint (AdBot|AppMetrica).

Yandex Mediation (5 networks)

Optional adapters: VK Ads (myTarget), Mintegral, BIGO Ads, Liftoff (Vungle), InMobi. Each has a toggle under Project Settings → Yandex Mediation (Android). Enable the network in the Yandex Advertising Network partner cabinet, then rebuild the APK. No extra Blueprint code.

AppMetrica Push (optional)

When Enable AppMetrica Push is on, the APK includes FCM/HMS providers. See Push: Setup.

Get Started

Get Started

Requirements

  • Unreal Engine 5.x
  • Android target platform configured in project
  • minSdkVersion 21 or higher
  • Account in Yandex Advertising Network

1. Add the Plugin

Copy the AdBot plugin folder into your project's Plugins/ directory:

YourProject/
  Plugins/
    AdBot/          ← paste here
      AdBot.uplugin
      Source/
      Platforms/

Restart the editor. In Edit → Plugins, find AdBot under the Advertising category and enable it.

2. Configure Ad Unit IDs

Open Edit → Project Settings → Plugins → AdBot and enter your Yandex Advertising Network placement IDs:

Interstitial Unit IDPlacement ID for interstitial ads

App Open Ad Unit IDPlacement ID for app-open format

3. Initialize in Blueprint

Recommended chain: privacy → Activate AppMetrica (or Init AppMetrica at Startup) → Initialize Yandex Ads once per session (Game Instance Event Init), unless Init Yandex Ads SDK at Startup already ran on Android:

Privacy first If you use a consent dialog, call Set Yandex Ads User Consent and other privacy nodes before Initialize Yandex Ads (or before the first ad load if using startup init).

4. Package for Android

Build the project targeting Android. The plugin's UPL adds Yandex SDK and VK adapter Gradle dependencies automatically.

Privacy & Consent

AdBot exposes all Yandex SDK privacy controls as Blueprint nodes in the AdBot | Yandex Privacy category.

Call privacy nodes before Initialize Yandex Ads Privacy flags are applied to Yandex SDK before YandexAds.initialize. Calling them after initialization is supported but preferred before the async Initialize Yandex Ads node (or before startup init runs).

App Open

App Open Ads

App open ads appear when users launch or return to the app. They can be closed at any time and are served from the vertical-only format in the Yandex network.

Async node flow

  1. Configure App Open Ad Unit ID in Project Settings.
  2. After Initialize Yandex Ads (or startup init), call Load App Open.
  3. On Ad Loaded — pass App Open Ad into Show App Open.
  4. On Ad Dismissed — call Load App Open again before the next show.

App Open outputs

Load App Open and Show App Open expose App Open Ad and Error. The ad token is single-use; reload after dismiss or failed show.

PinWhen it fires
Ad LoadedCreative ready (Load App Open)
Ad ShownAd appeared on screen
Failed To ShowShow call failed
Ad DismissedUser closed the ad

Banner Ads

Banners are small ads displayed at the bottom of the screen without blocking content. AdBot supports sticky and adaptive inline sizes.

Auto-load at startup

Only when Auto Load Banner on SDK Init is on (and a banner unit ID is set), Android loads the first banner using Default Banner Size Preset without Blueprint Load Banner. With the toggle off (default), call Load Banner after SDK init — Load Banner → Banner Size always overrides the preset.

No-Fill Auto-Retry

When Yandex returns empty inventory, the plugin automatically retries. Configure retries and delay in Project Settings (Banner No-Fill Max Retries / Banner No-Fill Retry Delay Seconds).

Blueprint outputs

Load Banner outputs Banner View and Error. Save Banner View in a Blueprint variable and pass it into Show Banner. Show Banner returns the same Banner View plus Error on validation failures.

Interstitial

Interstitial Ads

Full-screen ads displayed during natural breaks in the app, such as between game levels. Users can close them and return to the game.

Async node flow

  1. Call Load Interstitial (async).
  2. On Ad Loaded — pass Interstitial Ad into Show Interstitial at a natural pause.
  3. On Ad Dismissed — call Load Interstitial again before the next show.

Rewarded Video

Rewarded Video

Users voluntarily watch a rewarded video to earn in-game currency, extra lives, or other bonuses. AdBot exposes Load Rewarded Video and Show Rewarded Video async nodes (category AdBot | Rewarded).

Recommended order on Android Initialize SDK → Load Banner → Show Banner (wait for banner impression) → Load Rewarded Video → on Ad Loaded, Show Rewarded Video. This primes the WebView pipeline used by Unreal GameActivity.

Project Settings

Set Rewarded Ad Unit ID (demo-rewarded-yandex for testing, or your R-M-… block for production). Rebuild the APK after changing unit IDs.

Async node flow

  1. Add Load Rewarded Video. Optional Context Tags (targeting keywords for this request).
  2. On Ad Loaded — pass the Video output into Show Rewarded Video.
  3. On Failed To Load — read the Error structure (Code, Message, Domain).
  4. On Reward Granted — grant RewardType / RewardCount to the player.
  5. On Ad Dismissed or Failed To Show — call Load Rewarded Video again before the next show (one creative = one show).

Outputs of Load Rewarded Video

Both Ad Loaded and Failed To Load expose the same data pins: Video and Error. On success, Error.bHasError is false. On failure, Video is empty and Error contains the Yandex load error.

PinDescription
VideoBlueprint token for the loaded Yandex rewarded creative. Wire it into Show Rewarded Video.
ErrorFAdBotAdError from Yandex AdRequestError (description + code).

Outputs of Show Rewarded Video

All exec outputs expose the same data pins: Video, RewardType, RewardCount, and Error. Reward fields are filled on Reward Granted; Error is filled on Failed To Show.

PinDescription
VideoThe rewarded video token used by this show request.
Reward GrantedUser earned the reward (RewardType, RewardCount) from Yandex Reward.
Ad ImpressionImpression recorded by the SDK
Ad ShownAd appeared on screen
Ad DismissedUser closed the ad — reload before next show
Failed To ShowShow could not start — read the structured Error pin and offer a fallback.
Where the data comes from Reward.getType() → RewardType, Reward.getAmount() → RewardCount, and Yandex AdRequestError/AdError → FAdBotAdError.
Context tags Tags on Load Rewarded Video merge with the global targeting snapshot (Set Yandex Ad Targeting Context Tags). Snapshot tags come first; per-load tags are appended without duplicates.

AppMetrica Analytics

AppMetrica 8 (mobile analytics)

AppMetrica is Yandex's analytics SDK for Android/iOS. It is not the same as web Yandex.Metrica (site counter). AdBot bundles io.appmetrica.analytics:analytics:8.2.0 and exposes Blueprint nodes under AdBot|AppMetrica.

Why it matters for ads Yandex Mobile Ads SDK 8 expects AppMetrica to be present. Activating AppMetrica before YandexAds.initialize reduces banner Invalid SDK state and improves fill on production R-M-… placements.
One AppMetrica per app Do not enable the old marketplace YandexAppMetrica plugin (mobmetricalib 5.x) together with AdBot.

Blueprint nodes

Activate AppMetricaUses Project Settings (API key, session timeout, logs, tracking flags). Idempotent.

Android notification permission (POST_NOTIFICATIONS)

Cannot enable notifications “by default” on Android 13+ After install, Settings → Apps → [your app] → Notifications stays off until the user taps Allow in the system dialog. Declaring POST_NOTIFICATIONS in the manifest is not enough. This is required by Google — not an AdBot bug.

Starting with Android 13 (API level 33), apps must explicitly request a runtime permission to post notifications:

android.permission.POST_NOTIFICATIONS

For Android 12L (API 32) or lower, notifications are usually allowed at install unless the user disabled them manually.

What AdBot does automatically When Enable AppMetrica Push is ON, AdBot_UPL.xml adds POST_NOTIFICATIONS to the packaged manifest. AdBot may also call Request Post Notifications Permission after push activates. You should still call it from Blueprint on the main screen (first launch).
UE project: Extra Permissions Add +ExtraPermissions=android.permission.POST_NOTIFICATIONS in Project Settings → Android or DefaultEngine.ini in addition to AdBot manifest injection.
Android versionManifestRuntime dialog
API 33+ (Android 13+)Added by AdBot when push is enabledYes — Request Post Notifications Permission
API 32 and belowNot required for postingNo — node is a no-op unless user disabled notifications in settings

Reference: Android — Notification runtime permission.

When the game is not running

Yes — notifications can appear in the Android tray while Unreal is closed. FCM/HMS and AppMetrica Push deliver them natively; Blueprint is not involved in showing the icon.

Blueprint is required to register the device on first launch, request permission on Android 13+, and handle payload inside the game. See Blueprint: Push for three ready-made graphs (startup, cold start, foreground).

Runtime (Blueprint order)

  1. Rebuild APK with Enable AppMetrica Push = ON (Firebase fields baked into manifest at build time).
  2. Activate AppMetrica (or Init AppMetrica at Startup).
  3. Load Banner at least once successfully if using Init AppMetrica Push at Startup (push waits for banner onAdLoaded).
  4. Request Post Notifications Permission (Android 13+; on main screen after consent UI).
  5. Activate AppMetrica Push (or rely on deferred startup if Init AppMetrica Push at Startup is on).
  6. Bind Push Payload Received on Game Instance.

Logcat: adb logcat -s AdBotPush:I YaAdsBridge:I. Look for manifest_meta appIdLen=>0, AppMetrica Push activated. If the user denies permission, notifications stay off until enabled in system settings.

Tips & troubleshooting

Tips & troubleshooting

Practical notes for Yandex Mobile Ads 8.0.0 on Unreal GameActivity (Android).

Rewarded video — best results

  • Show a banner first in the session, then load/show rewarded (see Rewarded Video).
  • Use demo-rewarded-yandex while developing; switch to your R-M-… block for release and rebuild the APK.
  • If load returns no ads available, check block moderation and inventory in the Yandex partner cabinet.
  • Avoid HTTPS MITM (Charles/Fiddler) on device builds that play video ads — it can break VAST.

Banner — Invalid SDK state (mitigated)

On cold start with demo-banner-yandex, BannerAdView.loadAd could return Invalid SDK state. Mitigations in plugin: WebView warmup, deferred first load, AppMetrica activate, preset 90 dp (index 7), retries. Prefer production R-M-… banner ID for release.

Interstitial — slow first show

First interstitial may appear ~20–35 s after launch while the banner priming pipeline runs (by design to avoid parallel WebView loaders). Subsequent interstitials load faster.

FAQ

Frequently Asked Questions

Ads don't show on device — "Invalid SDK state" in Logcat

Enable SDK logging (Set Yandex Ads SDK Logging Enabled before Initialize Yandex Ads), open Logcat and search for YandexAds / LogAdBot. Common causes: AppMetrica key not set, calling Load Banner before Ad Data Loaded, or parallel fullscreen loads right after init.

Can I use this plugin without the AppMetrica key?

Yes. Ads may work without a key, but Yandex Mobile Ads SDK 8 requires AppMetrica SDK to be present. Leave the key empty and monitor Logcat for any warnings. For production, provide a real AppMetrica API key.

Does this plugin support iOS?

No. The plugin targets Android only via the Yandex Mobile Ads Android SDK. All JNI/UPL code is Android-specific. Blueprint nodes are safe to call on non-Android platforms (they do nothing).

Which Yandex SDK version is included?

The plugin bundles mobileads 8.0.0 (local Maven AAR) and the VK adapter mobileads-mytarget 5.45.3.1.

How do I test ads without a real Yandex account?

Use the demo placement IDs:
demo-banner-yandex, demo-interstitial-yandex, demo-rewarded-yandex, demo-appopenad-yandex

How do I show rewarded video?

Use Load Rewarded Video → Ad Loaded → Show Rewarded Video. Set Rewarded Ad Unit ID in Project Settings. On Android, show a banner earlier in the same session before the first rewarded load. Full flow: Blueprint: Rewarded Video.

AppMetrica vs Yandex.Metrica (web)

AdBot uses AppMetrica (mobile SDK). Web site counters (Yandex.Metrica) are a different product — use AppMetrica cabinet for game events.

Wrong banner height (50 dp instead of 90 dp)

Use enum index 7 = Inline max height 90 dp (8th item in the list). Index 6 is 50 dp. Set Default Banner Size Preset in Project Settings or pass the correct enum to Load Banner.

Notifications disabled after install (Android 13+)

Expected. Notifications cannot be on by default — call Request Post Notifications Permission and the user must tap Allow. Until then, system settings show notifications off. See POST_NOTIFICATIONS and What changed.

Push notifications do not arrive (but statistics work)

Statistics (AppMetrica) and push are separate. Check: (1) settings in Config/DefaultAdBot.ini, not DefaultGame.ini; (2) Enable AppMetrica Push ON and APK rebuilt after Firebase field changes; (3) FCM JSON in AppMetrica cabinet; (4) manifest_meta appIdLen>0 in logcat; (5) banner reached onAdLoaded if using Init AppMetrica Push at Startup; (6) Android 13+ permission granted; (7) adb logcat -s AdBotPush:I. Full checklist: Push setup.
