Plugin Details

Gaussian Splat Importer

Editor plugin for Unreal Engine 5.7 that imports, streams, edits, and renders 3D Gaussian splats inside LingvoBot and compatible projects. Supports local files (PLY / SPZ / SOG), native Virtual Page containers (.gsrad), optional World Labs Marble cloud generation, in-viewport Splat Edit tools, and 4DGS frame sequences with Sequencer integration.

📦

Multi-format import

PLY, SPZ, SOG; compressed bake-on-import or keep-on-disk decode.

🌐

Virtual Pages

SplatTree streaming for 100M+ splat city-scale scenes from .gsrad.

🎮

Custom GPU path

Instanced splat renderer with radix depth sort, frustum cull, optional SH lighting.

✂️

Splat Edit

Rect/lasso/box selection, delete, lock, move/rotate/scale with UE gizmo.

🎬

4DGS playback

Frame sequences (.ply/.sog/.spz), morph blend, Sequencer track.

World Labs

⚙️

Hardware presets

Features

Core capabilities

  • GaussianSplatActor — level actor with UGaussianSplatRenderComponent, quality presets (Low → Ultra), optional Virtual Page manifest.
  • Automatic Virtual Page preparation — large imports (> threshold splats) build a SplatTree + .gsrad sidecar for disk streaming.
  • Hardware-aware budgets — RAM/VRAM reserves, GPU pool auto-sizing, TDR-safe processing caps.
  • Occlusion culling — splats behind opaque UE geometry rejected before sort/draw (configurable bias).
  • Live stats overlay — resident splat count, Virtual Page scheduler metrics, sort mode.
  • Niagara fallback — CPU or GPU sim systems for compatibility; optional MLSLabs Pro path.
  • Sequencer — UMovieSceneGaussianSplatTrack drives 4DGS component time.

Not supported (v1)

  • Single-file .sog4d / TRBF checkpoint import
  • Save As on Virtual Page / streaming-only actors (splats not fully in RAM)
  • 4DGS sequence export from editor

Get Started

Installation & first scene

  1. Enable plugins Gaussian Splat Importer and Niagara in your .uproject.
  2. Rebuild the editor after C++ changes: Build.bat LingvoBotEditor Win64 Development.
  3. Open Project Settings → Plugins → Gaussian Splat Importer.
  4. Set Virtual Page Profile to Auto (Recommended) — or pick a tier matching your GPU.
  5. For World Labs cloud generation, enter your WLT-Api-Key.
  6. Open Tools → Gaussian Splat Importer → Import File and choose .ply, .spz, or .sog.
  7. Select the spawned GaussianSplatActor in Outliner; press F to frame the camera.
  8. For editing: Tools → Edit Gaussian Splats, select splats, use toolbar tools.

Niagara content (fallback renderer)

Create under Content/Materials/ (see plugin README):

  • NS_GaussianSplat_C — CPU Sim, NDI Gaussian Splat Data
  • NS_GaussianSplat_G — GPU Compute Sim, NDI Gaussian Splat Data (GPU)
  • M_GaussianSplat — Unlit Translucent material on Sprite Renderer; use Linear Color for vertex color (SH is linear)

Bind user parameters User.GaussianSplatData and User.SplatCount in both systems. Assign Niagara systems in Project Settings → Rendering.

After C++ or shader changes: full editor rebuild + restart (not Live Coding only). Hot-reload leaves stale DLLs and missing log lines.

GSPL v3 virtual page payload

Large scenes use quantized page bytes (position uint16, packed scales/rotations, optional proxy pages). Proxy spatial sampling picks the most significant splats per 3D grid cell instead of uniform decimation.

Virtual Pages

SplatTree streaming

Virtual Pages let city-scale scenes (10M–100M+ splats) render from disk without loading everything into RAM or VRAM.

Data model

  • SplatTree — hierarchical nodes with geometric error; each node has a proxy page (coarse) and optional leaf pages (full detail).
  • .gsrad file — binary page store (up to 64K splats per page, ~80 bytes/splat GPU pack).
  • Manifest asset — bounds, page directory, tree nodes, version metadata.

Runtime behavior

  1. Camera position + velocity drive visible cut selection (projected error threshold).
  2. Missing pages are read asynchronously from disk (MaxInFlightReadMB cap).
  3. Decoded pages upload to a fixed GPU page pool (auto-sized from VRAM fraction).
  4. Eviction reuses pool slots; ProxyRetainFrames protects fresh proxy from immediate eviction.
  5. Only active pages participate in depth sort and draw — typically 1–4M splats visible, not the full scene count.

Why VRAM stays ~3–5 GB on a 12 GB GPU

This is by design. The profile reserves VRAM for UE rendering, then allocates a fraction (e.g. 25–30%) to the splat page pool. The rest of the scene lives on disk and in CPU RAM cache. Task Manager Dedicated GPU Memory will not show 12 GB filled unless you force Custom settings with very high pool fraction (risk of OOM).

Debug overlay

Enable Virtual Page Debug Overlay in Project Settings. Shows activePages, activeSplats, poolPages, upload/read counters. Log line:

LogGaussianSplatImporter: Virtual Pages GPU publish: activePages=N activeSplats=M poolPages=P ...

Build Virtual Pages (manual)

Automatic preparation runs when splat count exceeds Auto Prepare Threshold (default 5M). For manual rebuild: use actor Details or importer pipeline after large PLY import.

GPU Rendering

Render pipeline

  • Instanced quads per splat — vertex shader projects 3D Gaussian to screen-space ellipse.
  • Radix sort back-to-front per view (r.GaussianSplat.SortMode).
  • Frustum + screen-radius cull before sort (gs.FrustumCullRadiusScale).
  • SH view-dependent color optional (gs.Render.UseSHViewColor).
  • Opaque occlusion — samples scene depth to drop splats behind UE meshes.
  • Cluster LOD path (optional Nanite-style clusters) for non-virtual static scenes.

Per-actor quality (EGaussianSplatQuality)

PresetEffect
Low / Medium / High / UltraDraw budget, sort rate, SH bands — independent from Virtual Page hardware profile

Global Splat Budget in Project Settings overrides all actors when > 0.

4DGS & Sequencer

Animated splat sequences

Add UGaussianSplat4DGSComponent to an actor. Load a folder of per-frame files.

Setup

  1. Add component → set Sequence Folder Path and File Pattern (Frame_{0000}.sog recommended).
  2. Enable Load Sequence Now or call LoadSequence from Blueprint.
  3. Optional: enable Frame Blend when .gsorder sidecar exists (from 4DGS→SOG tool).
  4. Create Level Sequence → bind actor → add track 4DGS Gaussian Splat.

Playback modes

ModeWhen
FlipbookDefault — discrete frame swap at FPS
Frame morphbEnableFrameBlend + .gsorder — lerps position, scale, rotation, opacity, SH
GPU deformgs.4DGS.UseGpuDeform=1 — compute shader morph path
Frame morph requires stable topology (same N gaussians deforming). Splats appearing/disappearing between frames need future correspondence support.

FAQ & Troubleshooting

Common issues

Black viewport / Splats: 0 with Virtual Pages

Cut stuck on root proxy only. Check log for activePages=1 activeSplats=8192. Fix: use Auto or a lower profile; restart editor after DLL rebuild; verify poolPages ≥ 64.

Low VRAM usage on high preset

Expected for streaming. Only visible pages occupy VRAM (~3–5 GB on 12 GB cards). Full scene is on disk. Pick profile matching your GPU, not Maximum · 32 GB GPU on a 12 GB card.

Build failed: Live Coding active

Close editor or press Ctrl+Alt+F11 before Build.bat.

Save As greyed out

Actor must have splats in RAM. Virtual Page streaming actors cannot Save As until edited splats are materialized.

Export / import orientation wrong

PLY uses mirror −1 X vs UE coordinates. Plugin applies consistent transform on import/export.

4DGS flicker between frames

Convert sequence with 4DGS → SOG for shared Morton order; enable Frame Blend + .gsorder sidecar.

Performance: sort lag on yaw

Try r.GaussianSplat.SortEveryNthFrame=2 or lower actor quality preset; for VP scenes ensure prefetch is not thrashing pool (PrefetchMaxPages).

Log files

  • Saved/Logs/LingvoBot.log — search LogGaussianSplatImporter, Virtual Pages GPU publish
  • UBT build log: %LOCALAPPDATA%/UnrealBuildTool/Log.txt

Third-party (bundled)

zlib, libwebp, spz under Plugins/GaussianSplatImporter/ThirdParty/. No npm/Python required for editor workflows.
