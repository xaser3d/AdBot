Android SDK distribution model

AdBot is an integration layer for Unreal Engine. It does not redistribute third-party ad or analytics SDK binaries inside the Fab/marketplace plugin package.

  • In the plugin ZIP: C++ and Blueprint API, AdBot_UPL.xml, Java bridge sources, Gradle/Maven coordinates, documentation.
  • At Android package time: Gradle downloads SDK artifacts from vendor Maven repositories on the game developer’s machine when building an APK/AAB.
  • Legal acceptance: Each developer accepts Yandex, Google, Huawei, and (if enabled) mediation network terms when using those SDKs in their shipped app.
Yandex Mobile Ads SDK License (Section 3.2–3.3)

Section 3.2 prohibits distributing the Yandex Mobile Ads SDK itself. AdBot complies by not bundling SDK binaries in the marketplace product. Section 3.3 allows distributing your application built with the SDK if you include the attribution: “Developed with the use of Yandex Mobile Ads SDK” (About screen, store listing, installer description file).

Full agreement: yandex.com/legal/mobileads_sdk_agreement

Mintegral (mediation)

When Enable Mintegral is ON, Gradle resolves Yandex’s adapter mobileads-mintegral:17.0.41.0. The Mintegral SDK may be pulled as a transitive dependency. AdBot does not ship the Mintegral SDK in the plugin ZIP. You need Yandex mediation setup plus Mintegral’s Terms of Service / SDK license (not only the Privacy Policy): Terms of Service, Privacy Policy.
