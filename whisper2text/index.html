Plugin Details

Whisper2Text

Offline speech-to-text for Unreal Engine 5.7 on Android and Win64, powered by whisper.cpp. No cloud API — models download once, inference runs on device.

🎤

Microphone capture

16 kHz mono PCM via async Start Voice Capture and Stop Recording.

📥

Model download

Tiny, Base, Small from Hugging Face (configurable URLs).

🔵

Blueprint async

Download Model, Initialize Whisper, Transcribe buffer or WAV file.

🌐

Languages

auto, en, ru, and other Whisper language codes.

⚙️

Auto / Manual mode

Auto picks the best downloaded model; Manual fixes Tiny/Base/Small.

📦

Fab-ready

Models are not shipped in the plugin ZIP — downloaded at runtime.

Subsystem UWhisper2TextSubsystem (Game Instance) coordinates models, engine, and capture. Log category: LogWhisper2Text.

Get Started

Quick start (Blueprint)

  1. Copy Plugins/Whisper2Text into your UE 5.7 project.
  2. Edit → Plugins — enable Whisper2Text (Win64 Editor + target platform).
  3. On first run: Download Model → Tiny (see Models & Download for URLs and manual install).
  4. Initialize Whisper (async).
  5. List Input Devices → Set Active Input Device (index 0).
  6. Start Voice Capture → speak → Stop Recording → PCM buffer.
  7. Transcribe Audio Buffer (Language: auto, en, or ru) → print Transcribed string.
Internet required once for model download. After that, transcription works offline.

Requirements

  • Unreal Engine 5.7
  • Windows Editor (Win64) for plugin UI and Blueprint authoring
  • Packaged targets: Win64 or Android (ARM64)
  • Microphone permission on Android (RECORD_AUDIO added by plugin UPL)
  • Disk space for models: Tiny ~39 MB, Base ~74 MB, Small ~244 MB (ggml-*.bin from Hugging Face)

Models & Download

Whisper models

Models are not shipped inside the plugin ZIP. You download them once at runtime (Blueprint) or manually from Hugging Face.

SizeFileApprox. sizeUse case
Tinyggml-tiny.bin~39 MBFastest — first test on mobile and desktop
Baseggml-base.bin~74 MBBetter accuracy, still reasonable on mid-range phones
Smallggml-small.bin~244 MBHigher quality; slower on weak devices

How to download (Blueprint — recommended)

  1. Run PIE or a packaged build with internet access.
  2. Add async node Download Model (Whisper2Text | Models) → pick Tiny, Base, or Small.
  3. Wait for pin Completed (optional: wire Progress to a UI bar).
  4. Call Initialize Whisper. Check with Is Model Downloaded if needed.

How to download (Project Settings — Editor)

Edit → Project Settings → Plugins → Whisper2Text → Models — use the Download button next to Tiny / Base / Small. Progress bar and status appear in the panel. Files save to Saved/Whisper2Text/Models/ in your project (same path as runtime download on Win64).

Project Settings → Model Download Url Tiny is the direct file URL the plugin requests in step 2 — not a “download page”. You normally do not open it in a browser unless downloading manually (see below).

Default download URLs (Hugging Face)

Pre-filled in Config/DefaultWhisper2Text.ini and Edit → Project Settings → Plugins → Whisper2Text → Models:

SettingDefault URL
Model Download Url Tinyggml-tiny.bin
Model Download Url Baseggml-base.bin
Model Download Url Smallggml-small.bin

Full catalog (quantized variants, .en models, etc.): huggingface.co/ggerganov/whisper.cpp. If you use another file, update the matching URL in Project Settings and use the exact filename the plugin expects (ggml-tiny.bin by default for Tiny).

Manual download (browser)

  1. Open one of the links above (or the Hugging Face file page) and save the .bin file.
  2. Copy it to the models folder with the exact name (e.g. ggml-tiny.bin):
  • Win64 / Editor: <YourProject>/Saved/Whisper2Text/Models/ggml-tiny.bin
  • Android: app persistent download dir → Whisper2Text/Models/ (same filename)

The file must be larger than 1 MB or the plugin treats it as missing. Then run Initialize Whisper (no Download node required).

Storage paths

  • Win64: <Project>/Saved/Whisper2Text/Models/
  • Android: persistent app download directory (survives reinstall depending on OS)

Models are not included in the Fab/marketplace ZIP. Use Download Model in Blueprint, or manual copy (see above).

Recognition modes

  • Auto — Initialize Whisper loads the best already downloaded model: Small → Base → Tiny. On low-RAM Android, Project Settings can force Tiny preference.
  • Manual — call Set Recognition Mode = Manual, Set Model Size, then Initialize Whisper.

Android

Android notes

  • Target ARM64 (arm64-v8a).
  • Plugin UPL adds RECORD_AUDIO — request runtime mic permission in your game if needed.
  • JNI capture: com.whisper2text.AudioCaptureBridge (16 kHz mono).
  • Test with Tiny or Base first; avoid Small on low-end devices until profiled.
  • Long recordings use CPU and battery — respect Max Capture Duration.

FAQ & Tips

FAQ

Initialize Whisper fails

Ensure the model finished downloading (Is Model Downloaded). Check Output Log for LogWhisper2Text.

Empty transcription

Verify PCM is 16 kHz mono float; capture actually ran; language code matches speech (try auto).

Download fails on device

Check HTTPS access to huggingface.co. Ensure Model Download Url Tiny (etc.) in Project Settings is not empty — it must be a direct .bin link, not a folder page. See Output Log: LogWhisper2Text. For offline installs, download in a browser and copy to Saved/Whisper2Text/Models/ (see Models & Download).

Where is Model Download Url Tiny?

Edit → Project Settings → Plugins → Whisper2Text → Models. It is the URL the plugin uses internally; open the link in a browser only for manual download.

Slow on Android

Use Tiny; reduce capture length; lower thread count if thermals spike; disable Small on <4 GB RAM devices.

Third party

whisper.cpp and ggml are bundled in the plugin source. See THIRD_PARTY_NOTICES.md in the plugin folder (MIT license).

Performance tips

  • Download only the model sizes you need.
  • Initialize once per session; reuse the same subsystem.
  • Do not run transcription on the game thread — async nodes already offload work.
